# Public Reference Surface Authority Standard v1

## Purpose

Define minimum visible authority requirements for Hoax.ai’s 41-route public reference surface after Sprint 95 repairs.

## Page Role Labels

Every public page must make its role clear. Legacy and support pages use `Page role:` labels such as Boundary Reference, Governance Reference, Standard Reference, Protocol Reference, Registry Reference, Ontology Reference, or General Public Overview.

## System Placement

Legacy and support pages must include a **How this page fits Hoax.ai** section explaining layer placement and a **Hoax.ai System Navigation** block linking to utilities, concepts, pathways, and boundary references.

## Internal Authority Links

Legacy and support pages must link to at least one public utility or public reference route with contextual authority connection copy.

## Citation Safety

Public pages must avoid unnegated forbidden exact n-grams listed in governance validators. Boundary language must not imply truth verification, automated assessment, or binary authenticity label outputs.

## Metadata Requirements

Every public HTML page must have exactly one H1, a canonical URL, meta description, and Open Graph title and description.

## Consolidated Layer Requirements (Sprint 94)

Twenty-two utility, core reference, deep reference, and pathway pages retain page role labels, Hoax.ai System Navigation blocks, and IA Capsules.

## Homepage Requirements

Homepage must include Navigate Hoax.ai by Evidence-Risk Layer and Hoax.ai System Navigation.

## Prohibited Behaviors

Upload, scoring, verdict, detector claim, public API, automated report, JavaScript (except pre-existing JSON-LD), forms, chatbot, generator, and real-world case evaluation remain unauthorized.

## Enforcement

`validators/validate_public_reference_surface_authority_review_v1.py` and `validate_all.py`.
