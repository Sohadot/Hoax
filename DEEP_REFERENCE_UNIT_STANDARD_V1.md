# Deep Reference Unit Standard v1

## Deep Reference Unit Statement

Deep reference units are 900+ word static pages with full Sprint 86–90 component stacks and citation-safe boundaries.

## Required Page Structure

Reference summary, definition, why it matters, how it appears, signals, questions, what not to conclude, evidence posture relationship, Reference Answer, Source Confidence, Cite This Reference, Retrieval Capsule, Related Concepts, page-end navigation, boundary reminder.

## Minimum Content Depth

900 visible words minimum per deep reference route.

## Human Readability Requirements

Clear H1, thesis line, practical questions, forbidden-leap section, utility links.

## AI Readability Requirements

Stable anchors, Reference Answer, Retrieval Capsule, AI-readable link capsule.

## Citation/Retrieval Requirements

Canonical URL in cite block; page-specific boundary reminders; no truth-verification framing.

## Source-Confidence Requirements

Support type, can/cannot support, boundary, related reference on every deep page.

## Boundary Requirements

No binary authenticity labels, numeric certainty outputs, verdicts, uploads, or automated reports.

## Internal Link Requirements

Link to three utilities, three+ existing reference routes, two+ sibling new routes, homepage.

## Forbidden Patterns

Forbidden exact n-grams, forms, JavaScript, real-world cases, detector claims, scoring language.

## Thin-Page Prevention Rules

Reject routes below 900 words or missing required components.
