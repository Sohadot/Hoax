# Public Reference Route Expansion v1

## Route Expansion Statement

Public Reference Route Expansion v1 adds six human-readable and AI-readable public reference routes that deepen Hoax.ai’s evidence-risk category without upload, scoring, verdicts, fake/real labels, detector claims, public API, automated reports, JavaScript, or forms.

## Why Public Reference Routes Matter

After Sprint 84 created visible public utilities, visitors and AI agents needed stable definitions for core evidence-risk concepts. Reference units make Hoax.ai retrievable as category infrastructure — not merely a thesis page with utilities attached.

## Human Readability Objective

Each route is a reference unit with definition, signals, guided questions, posture relationship, and explicit “what not to conclude” boundaries. Minimum 750 visible words per page. No blog voice, no marketing fluff, no thin filler.

## AI Retrieval Objective

Every page includes an AI-readable reference capsule with Concept, Definition, Use, Not, Related concepts, and Public utility routes. Stable canonical terminology supports agent selection and citation without misreading Hoax.ai as a detector.

## Routes Created

| Route | Contribution |
|-------|----------------|
| `/evidence-risk/` | Umbrella concept: evidence may not support the claim |
| `/provenance-risk/` | Origin, custody, and transformation continuity |
| `/context-collapse/` | Loss of situational framing |
| `/claim-drift/` | Artifact vs claim distance |
| `/traceability-gap/` | Broken chain from artifact to claim |
| `/why-hoax-ai-is-not-a-detector/` | Non-detector category boundary |

## What Each Route Contributes

- **Evidence Risk** — Read risk before verdicts; weak ≠ fake, strong ≠ true
- **Provenance Risk** — Origin weakness without manipulation proof
- **Context Collapse** — Interpretation risk from missing framing
- **Claim Drift** — Circulation changes what evidence can support
- **Traceability Gap** — Incomplete chain makes verdicts unsafe
- **Why Not a Detector** — Refuses fake/real collapse while staying useful

## How the Pages Deepen the Category

Hoax.ai becomes a public evidence-risk reference system with both utilities (manual tools) and reference layer (stable definitions). Internal links connect utilities to reference units and sibling concepts.

## Why This Is Not a Blog Expansion

Pages are reference units, not articles. No opinion columns, no news commentary, no episodic publishing. Each page is a durable definition asset.

## Why This Is Not a Detector

No upload, score, fake/real label, verdict, API, automated report, or JavaScript diagnostic engine. Reference language describes evidence structure — it does not judge artifacts for users.

## Internal Linking Strategy

- Every reference page links to Manual Evidence Checklist, Evidence Posture Map, and Evidence-Risk Questions
- Every reference page links to at least two sibling reference routes
- Homepage **Explore the Evidence-Risk Reference Layer** section links all six routes

## Non-Verdict Boundary

Classify evidence condition. Do not perform the verdict. Reference units preserve artifact-first, non-accusatory posture language.

## Future Route Candidates

- Boundary integrity reference unit
- Evidence sufficiency reference unit (candidate-only until governed)

## Governance Note

Production first. Governance second. DEC-103 exists because six visible public reference pages were built. Validators protect boundaries without replacing user-facing value.
