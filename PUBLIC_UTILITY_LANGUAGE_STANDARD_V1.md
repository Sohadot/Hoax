# Public Utility Language Standard v1

## Standard Statement

Public utility language describes what Hoax.ai helps people do with evidence risk — manually, conceptually, and without verdict output. This standard governs homepage copy, README positioning, and future public utility pages.

## Allowed Public Language

- helps people read evidence risk
- manual evidence-risk utility
- evidence posture reference
- guided evidence questions
- non-verdict evidence map
- synthetic media risk literacy
- public evidence-risk reference system
- manual evidence posture guide
- maps evidence risk manually and conceptually
- teaches when evidence becomes judgeable, or not
- does not judge artifacts for users
- helps people ask better evidence questions

## Forbidden Public Language

- detects fake media
- verifies truth
- scores authenticity
- proves manipulation
- identifies deception
- certifies evidence
- confirms real or fake
- automated investigation
- AI detector
- upload your file
- scan for fakes
- truth score
- authenticity percentage
- deepfake detector
- fraud finder

## Negation Rule

Forbidden phrases may appear only inside explicit denial contexts (e.g. “not a fake/real detector”). Denial copy supports boundaries; it must not replace utility-first category language in hero or primary positioning.

## Utility vs Product Language

| Prefer | Avoid |
|--------|-------|
| reference system | platform |
| manual guide | automated tool |
| evidence map | detection dashboard |
| guided questions | classifier output |
| risk literacy | truth verification |

## Application Scope

This standard applies to index.html, README.md, and all future Sprint-authorized public utility pages.
