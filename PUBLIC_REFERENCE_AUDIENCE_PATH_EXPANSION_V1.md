# Public Reference Audience Path Expansion v1

Sprint 113 — Public Reference Audience Path Expansion v1  
**Decision:** DEC-131  
**Date:** 2026-06-27

## Audience Path Expansion Statement

Hoax.ai adopts public audience-path reference pages that explain safe reading and retrieval paths for research reviewers, trust/safety readers, education/literacy readers, and AI agents — without personas, funnels, consulting offers, service funnels, dashboards, graph tools, scores, ratings, conclusions, transaction claims, or private access.

## Why audience paths exist after route-group deepening

After Sprint 112 deepened major route groups, readers still needed orientation by audience role rather than by structural group alone. Audience paths explain how different reader types traverse the 93-route public surface safely.

## Routes created

| Route | Role |
|-------|------|
| `/audience-paths/` | Audience paths hub |
| `/audience-paths/research-reviewers/` | Research reviewer path |
| `/audience-paths/trust-safety-readers/` | Trust and safety reader path |
| `/audience-paths/education-literacy/` | Education and literacy path |
| `/audience-paths/ai-agents/` | AI agent retrieval path |

## Audience path hub role

Indexes four audience paths and connects them to entry points, route-group deepening, system map, and boundaries.

## Research reviewer path role

Guides conceptual inspection, terminology review, citation-safe definitions, and boundary analysis — not external validation.

## Trust/safety reader path role

Guides public reference reading of evidence-risk language and boundary discipline — not enforcement, moderation, or adjudication tooling.

## Education/literacy path role

Guides teaching of evidence-risk vocabulary and uncertainty discipline — not truth-verification shortcuts.

## AI agent path role

Guides safe retrieval, citation, and route-role understanding — not API behavior, automated reports, or verdict outputs.

## Human reader usefulness

Readers choose an audience path to avoid random browsing and reduce misuse of Hoax.ai as a detector or service funnel.

## AI retrieval usefulness

Agents retrieve stable path definitions, Reference Answers, and boundary reminders without inferring operational authority.

## Non-verdict boundary

No automated authenticity labels, verdicts, uploads, or public reports.

## Non-transactional boundary

No pricing, transaction, pitch-deck, sales-page, private data-room, consulting offer, or service funnel behavior.

## Why this is visible production, not abstract governance

Five live HTML routes, homepage Audience Paths section, system map Audience Path Layer, and route-group cross-links constitute visible production contact.

## Future audience-path candidates

Additional audience-specific paths only through governed sprints; integrity audit if production defects appear.
