# Public Category Language Layer

## A. Purpose

Sprint 26 creates Hoax.ai’s first public category language layer. This layer makes the category legible by showing the core terms, relationships, boundaries, and non-verdict logic behind Hoax.ai before any public engine, classifier, upload workflow, or scoring interface exists.

## B. Non-purpose

This layer does not:

- create a public classifier;
- create a public engine;
- create a public tool;
- create upload functionality;
- create scoring;
- create fake/real outputs;
- create forms;
- create analytics;
- create API;
- create monetization;
- modify DNS or Cloudflare;
- launch the Hoax.ai custom domain;
- authorize broader reference expansion;
- authorize individual term pages.

## C. Governing Principle

A category language layer names the system of judgment before any tool performs work.

Hoax.ai must teach how to speak about evidence posture before it offers any public classifier.

## D. Category Language Definition

Category language is a governed public vocabulary and relation system that explains how Hoax.ai describes evidence condition, artifact boundaries, source confidence, provenance gaps, output limits, and non-verdict reasoning.

## E. Category Language Is Not

Category language is not:

- detection;
- classification output;
- scoring;
- verdict;
- truth certification;
- upload workflow;
- public engine;
- public classifier;
- software service;
- fact-checking judgment;
- subject accusation.

## G. Initial Relation Model

Required relations:

- Evidence Posture depends on Artifact–Subject Separation.
- Evidence Posture is informed by Source Confidence.
- Evidence Posture is constrained by Provenance Gap.
- Not Assessable protects against false certainty.
- Output Boundary limits what Hoax.ai may say.
- Claim and Source Traceability governs support discipline.
- Synthetic Fragility names the broader condition, but remains boundary-limited until refined.
