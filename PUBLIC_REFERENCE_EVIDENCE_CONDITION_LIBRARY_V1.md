# Public Reference Evidence Condition Library v1

## Evidence Condition Library Statement

Hoax.ai adopts Public Reference Evidence Condition Library v1: six public reference routes that define recurring evidence conditions readable before conclusions. The library is not a detector, not a verdict taxonomy, not a scoring system, and not an operational assessment tool.

## Why evidence conditions exist after audience paths

Sprint 113 added audience paths for reader-orientation. Sprint 114 adds condition-definition depth so humans and AI agents can name evidence states conceptually without collapsing them into truth labels, scores, or enforcement outputs.

## Routes created

- `/evidence-conditions/` — hub
- `/evidence-conditions/source-uncertainty/`
- `/evidence-conditions/provenance-discontinuity/`
- `/evidence-conditions/context-loss/`
- `/evidence-conditions/claim-evidence-misalignment/`
- `/evidence-conditions/traceability-break/`

## Evidence condition hub role

The hub orients readers across five condition pages and links to concepts, pathways, route groups, audience paths, and system map layers without creating a closed classification system.

## Source uncertainty role

Defines unclear origin or authorship as a condition — not proof of manipulation or falsehood.

## Provenance discontinuity role

Defines gaps in provenance chains as a condition — not proof of falsification.

## Context loss role

Defines missing or collapsed context as a condition — not proof of deception.

## Claim-evidence misalignment role

Defines claims that exceed available evidence as a condition — not a truth status label.

## Traceability break role

Defines broken traceability paths as a condition — often compatible with not-assessable posture, not a verdict.

## Human reader usefulness

Humans gain vocabulary to describe evidence posture before conclusions, linked to manual utilities and pathway language.

## AI retrieval usefulness

AI agents retrieve stable condition definitions, Reference Answer blocks, and Retrieval Capsules without inferring case outcomes or operational mandates.

## Non-verdict boundary

No binary authenticity labels, numeric certainty outputs, verdicts, upload capability, or public reports.

## Non-transactional boundary

Not a transaction page, pricing statement, acquisition term document, representative mandate, legal representation, financial representation, private due-diligence room, downloadable-report, pitch-deck, sales-page, score-card, rating-system, dashboard, or graph tool.

## Why this is visible production, not abstract governance

DEC-132 governs six static HTML pages on the 99-route public surface with homepage and system map links built in Sprint 114.

## Future evidence condition candidates

Future conditions require explicit governance gates; the library is intentionally non-exhaustive.
