# Public Reference Retrieval Index Standard v1

**Decision:** DEC-136  
**Sprint:** Sprint 118  
**Status:** Active standard

## Standard rule

Public retrieval indexes may organize Hoax.ai routes by retrieval intent, but must not create search behavior, ranking, scoring, generated answers, verification, detection, case assessment, operational review, workflow behavior, dashboard behavior, API behavior, upload behavior, report generation, or transaction surfaces.

## Route constraints

- Exactly one Sprint 118 route: `/retrieval-index/`.
- No child routes under `/retrieval-index/`.
- Static HTML only.

## Technical constraints

- No JavaScript.
- No forms.
- No uploads.
- No API behavior.
- No search box.
- No live filtering.
- No generated-answer layer.
- No dashboard, graph-tool, score-card, or rating-system behavior.

## Content constraints

- At least 2,000 visible words.
- Exactly one H1.
- Canonical/meta/OG tags present.
- Human-readable route orientation.
- AI-retrieval-readable route orientation.

## Required retrieval-intent groups

1. Thesis and category framing
2. Evidence-risk concepts
3. Evidence condition definitions
4. Crosswalk and relation retrieval
5. Reading sequence orientation
6. Audience-specific orientation
7. Boundary and non-detector language
8. Citation and retrieval blocks
9. Strategic review and category-asset understanding
10. System-level navigation

## Required sections

The route must include all 27 required sections listed in Sprint 118.

## Linking constraints

The route must link to key companion pages, including homepage, system map, reading sequences, evidence conditions hub, condition pages, crosswalk, concept pages, route groups, audience paths, and strategic/reviewer routes.

## Non-transactional boundary

The route cannot present pricing, transaction, sales, consulting, legal representation, or financial representation claims.
