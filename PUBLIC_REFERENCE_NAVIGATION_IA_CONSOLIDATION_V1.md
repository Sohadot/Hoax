# Public Reference Navigation and IA Consolidation v1

## Navigation IA Statement

Sprint 94 consolidates visible information architecture across Hoax.ai’s 41-route public surface so utilities, core references, deep references, pathways, and boundary pages read as one coherent evidence-risk system.

**Decision:** DEC-112 | **Sprint:** 94 | **Routes:** 41 (unchanged)

## Why Navigation IA Matters

After pathway pages arrived in Sprint 93, visitors needed clearer route groups, page roles, and system navigation — not more pages. Navigation must reveal the evidence-risk system, not merely list URLs.

## Route Groups Defined

1. **Public Utilities** — manual application layer
2. **Core Reference Concepts** — first-layer definitions
3. **Deep Reference Concepts** — precise category language
4. **Evidence-Risk Pathways** — condition-based journeys
5. **Boundary and Standard References** — non-verdict and non-detector boundaries

## Visible Improvements

- Homepage section: Navigate Hoax.ai by Evidence-Risk Layer
- Page role labels on 22 utility/reference/pathway pages
- Hoax.ai System Navigation blocks on homepage and 22 pages
- IA Capsules on 22 pages
- Page-specific Where to go next guidance
- Pathway entry points on key reference pages

## Human Usability Role

Visitors can identify where to start and what to read next by layer and condition.

## AI Retrieval Role

IA Capsules expose page role, route group, upstream/downstream pages, and related pathways as static HTML for agent parsing.

## Why This Is Not Generic SEO

Improvements structure existing content — no keyword stubs or thin pages.

## Why This Is Not a Detector

No upload, score, verdict, API, report, JavaScript, or form behavior.

## Non-Verdict Boundary

Navigation language preserves evidence-condition reading before conclusions.

## Future IA Candidates

Surface authority review and selective navigation refinements require separate governed authorization (Sprint 95 candidate).
