# Public Reference Source Use Orientation Standard v1

**Decision:** DEC-138  
**Route:** `/source-use-orientation/`

## Standard scope

This standard governs the visible Sprint 122 source-use-orientation route only.

## Required production properties

- Static HTML with one H1
- Minimum 2,000 visible words
- Canonical URL, meta description, and Open Graph tags
- Twenty-six required sections with stable anchors
- Ten source-use orientation themes
- Reference Answer, Source Confidence, Cite This Reference, Retrieval Capsule, boundary reminder, and non-transactional review boundary blocks
- No JavaScript, forms, uploads, or API behavior

## Prohibited behaviors

- Proof claims and verification claims
- Authority certification and source endorsement
- Legal or academic source status claims
- Detector evidence and score basis claims
- Verdict support and case conclusion behavior
- Source database interface and source authority directory behavior
- Workflow, operational procedure, and report-generation behavior
- Dashboard, graph-tool, score-card, and rating-system behavior
- Transaction, sales-page, consulting-funnel, due-diligence-room, and legal/financial representation behavior

## Navigation requirements

Homepage, system map, retrieval index, citation orientation, and reading sequences must link to `/source-use-orientation/` where source-safe orientation is relevant.
