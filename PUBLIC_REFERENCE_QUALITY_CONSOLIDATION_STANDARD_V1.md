# Public Reference Quality Consolidation Standard v1

## Quality Standard Statement

Public reference quality requires visible coherence across metadata, components, boundaries, links, and retrieval language on all 29 routes.

## Metadata Standard

One H1, canonical URL, unique meta description, matching Open Graph title and description per route.

## Component Consistency Standard

Core utility/reference pages preserve Reference Answer, Source Confidence, Cite This Reference, Retrieval Capsule, and page-end reference navigation.

## Reference Answer Quality Standard

Canonical question and short answer must match page role; usage and do-not-use fields must differ.

## Source-Confidence Quality Standard

Support type must match page role; cannot-support must negate verdict and score behavior.

## Citation/Retrieval Quality Standard

Cite and retrieval blocks must include page-specific boundary reminders; must not imply verification or assessment.

## Internal Link Quality Standard

Homepage cards should deep-link to `#reference-answer` where applicable. Core pages must link to relevant utility and reference routes.

## Forbidden Regression Patterns

Reintroducing identical boundary boilerplate across core pages, colloquial fake/true OG framing, forbidden exact n-grams, missing anchors, or new tool behavior.

## Future Consolidation Rules

Inspect after each public-layer sprint. Repair visible regressions before adding governance. Keep sitemap at 29 unless expansion gate passes.
