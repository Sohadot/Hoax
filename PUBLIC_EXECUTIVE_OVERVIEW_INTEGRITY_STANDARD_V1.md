# Public Executive Overview Integrity Standard v1

Sprint 105 — Public Reference Executive Overview Integrity Audit v1  
**Decision:** DEC-123  
**Date:** 2026-06-26

## Executive Overview Integrity Standard Statement

Executive-overview integrity means the overview gives concise public orientation without creating conclusions, pitch claims, private access, transactions, reports, or operational claims.

## Executive overview route-count requirements

- Sitemap must remain at the current authorized public route count (73 after Sprint 104)
- Route registry must match sitemap count exactly
- public-file-registry must align with route HTML files if used
- Integrity audits must not add new public routes

## Executive overview component preservation requirements

Every executive-overview page must preserve Reference summary, Overview purpose, Executive reading path, What this overview supports, What this overview does not claim, Reference Answer, Source Confidence, Cite This Reference, Retrieval Capsule, Boundary reminder, Non-transactional review boundary, and page-end reference navigation.

## Executive overview metadata requirements

Every executive-overview page must have exactly one H1, canonical URL, title element, meta description, Open Graph title, and Open Graph description.

## Executive overview link requirements

Every executive-overview page must link to `/executive-overview/`, at least three strategic, external-review, or reviewer-packet routes, at least five reference, utility, or pathway routes, and at least two sibling executive-overview routes. Internal links must resolve to registered public routes.

## Executive overview role clarity requirements

Every executive-overview page must clearly state it is a public high-level reference overview and concise orientation layer — not a sales page, pitch deck, private data room, downloadable report, transaction surface, scorecard, or verdict system.

## Non-verdict requirements

Executive-overview pages must not imply automated authenticity labels, numeric certainty outputs, verdicts, uploads, detector behavior, or public automated reports.

## Non-transactional requirements

Executive-overview pages must not imply pricing statements, transaction pages, acquisition term documents, representative mandates, or legal or financial representation.

## Pitch-deck prohibition

Executive-overview pages must not function as pitch decks. Safe negative language (`not a pitch deck`) is required where role clarity is needed.

## Sales-page prohibition

Executive-overview pages must not function as sales pages. Safe negative language (`not a sales page`) is required where role clarity is needed.

## Private data-room prohibition

Executive-overview pages must not claim or imply private data-room access or private due-diligence room behavior.

## Downloadable report prohibition

Executive-overview pages must not claim or imply downloadable report behavior.

## Forbidden regression patterns

Forbidden positive phrases in public HTML include pitch deck, sales page, private data room, downloadable report, pricing statements, transaction pages, detector claims, upload behavior, and verdict language as capability claims.

## Future executive-overview audit rules

Future integrity audits must inspect the full public surface, preserve the Executive Overview Integrity Snapshot on the overview hub, check pitch-deck and sales-page drift, and record visible repairs before governance adoption.
