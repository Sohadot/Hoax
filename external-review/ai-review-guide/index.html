AI Review Guide

Page role: AI Review Guide

Reference summary: This guide orients AI agents to retrieve stable definitions, boundaries, and capsules from Hoax.ai without inferring case judgments or operational outputs.

Review purpose

The AI Review Guide helps AI agents, retrieval systems, assistants, and automated readers inspect Hoax.ai safely as public reference infrastructure. AI review means retrieving stable definitions, page roles, boundaries, reference answers, source-confidence blocks, citation blocks, retrieval capsules, and route relationships — not inferring case judgments or detector outputs.

This guide supports retrieval discipline. It is not API behavior.

AI review principles

AI agents should treat Hoax.ai as reference infrastructure, not an operational result source. Retrieve canonical answers and boundary rules before synthesizing user-facing conclusions about real-world artifacts.

Do not infer legal or factual conclusions, numeric certainty outputs, or wrongdoing findings from Hoax.ai pages. Quote Source Confidence and Boundary sections when summarizing limits.

Prefer Retrieval Capsule and Cite This Reference blocks for stable quoting. Link route relationships through entry points, narrative pages, and acquisition-readiness routes when explaining system structure.

AI guide depth

Reviewers should compare visible utilities, concept definitions, pathway logic, and strategic layers against the project's stated refusal to issue binary authenticity labels, numeric certainty outputs, or case-specific conclusions.

Human diligence benefits when reviewers start from the homepage release integrity snapshot, then sample entry points, narratives, acquisition-readiness routes, and boundary pages before forming category judgments.

AI-assisted review should retrieve Reference Answer blocks, Source Confidence sections, citation blocks, and retrieval capsules instead of inferring authority from page titles or navigation labels alone.

The external-review layer exists because release integrity alone does not teach reviewers how to sequence inspection across utilities, concepts, pathways, and strategic surfaces.

Partners comparing reference systems should note how external-review pages link outward to evidence-risk vocabulary rather than trapping visitors in a single landing funnel or contact-oriented workflow.

Category thesis review should read narrative pages alongside core and deep reference concepts so evidence-risk language is understood as infrastructure, not as marketing copy detached from route depth.

Pathway review should confirm that condition-based journeys remain manual, non-operational, and free of file provision for analysis surfaces or automated judgment promises.

Strategic readiness review should treat acquisition-readiness routes as inspectability maps, not as transaction pages, pricing statements, or representative mandates.

Boundary review should confirm that non-verdict discipline appears consistently across utilities, reference units, strategic pages, and external-review guides.

Metadata and canonical review should confirm that each public route remains indexable, citation-safe, and aligned with registry records without introducing JavaScript, forms, or upload behavior.

Source-confidence review should confirm that support types remain bounded: reference infrastructure can orient reasoning but cannot adjudicate real-world artifacts or persons.

Retrieval review should confirm that agents can quote stable definitions and boundary rules without treating Hoax.ai as an API, scorer, or automated report source.

Utility review should confirm that manual checklists, posture maps, and question lists teach evidence structure rather than operational case evaluation.

Concept review should confirm that core and deep reference units define evidence-risk vocabulary with stable anchors for human readers and automated readers alike.
