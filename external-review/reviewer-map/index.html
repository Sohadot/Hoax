Use this map when you need area-based orientation rather than a linear checklist or AI-specific retrieval guide.

Reviewer map depth

Future external-review expansion may add more inspection guides, but Sprint 101 authorizes only static reference pages with no chatbot behavior, generator behavior, or public API endpoints.

Reviewers documenting findings should cite canonical URLs and boundary reminders rather than paraphrasing Hoax.ai as a detector, verdict engine, or upload tool.

Inspection discipline matters because synthetic-media discourse often collapses evidence conditions into shareable conclusions; Hoax.ai's asset value is slowing that collapse through reference structure.

External review readiness therefore complements release integrity: one confirms the surface is whole, the other teaches how to walk it responsibly.

External review on Hoax.ai means walking the published reference system as a governed knowledge surface rather than treating any page as a hidden product demo or operational endpoint.

Reviewers should compare visible utilities, concept definitions, pathway logic, and strategic layers against the project's stated refusal to issue binary authenticity labels, numeric certainty outputs, or case-specific conclusions.

Human diligence benefits when reviewers start from the homepage release integrity snapshot, then sample entry points, narratives, acquisition-readiness routes, and boundary pages before forming category judgments.

AI-assisted review should retrieve Reference Answer blocks, Source Confidence sections, citation blocks, and retrieval capsules instead of inferring authority from page titles or navigation labels alone.

The external-review layer exists because release integrity alone does not teach reviewers how to sequence inspection across utilities, concepts, pathways, and strategic surfaces.

Partners comparing reference systems should note how external-review pages link outward to evidence-risk vocabulary rather than trapping visitors in a single landing funnel or contact-oriented workflow.

Category thesis review should read narrative pages alongside core and deep reference concepts so evidence-risk language is understood as infrastructure, not as marketing copy detached from route depth.

Pathway review should confirm that condition-based journeys remain manual, non-operational, and free of file provision for analysis surfaces or automated judgment promises.

Strategic readiness review should treat acquisition-readiness routes as inspectability maps, not as transaction pages, pricing statements, or representative mandates.

Boundary review should confirm that non-verdict discipline appears consistently across utilities, reference units, strategic pages, and external-review guides.
