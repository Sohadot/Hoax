# Public Evidence Condition Library Standard v1

## Evidence Condition Library Standard Statement

Evidence condition pages define readable public reference states without conclusions, scores, ratings, dashboards, detector outputs, private access, transactions, reports, consulting offers, or service funnels.

## Required page structure

Reference summary, condition definition, conceptual thesis, contextual frame, condition reading, human use, AI retrieval use, confusion prevention, supports/does-not-claim, relation to other conditions, Reference Answer, Source Confidence, Cite This Reference, Retrieval Capsule, strategic value, page-end navigation, boundary blocks.

## Minimum depth requirement

At least 1,400 visible words per new route.

## Required reference components

All components listed in `data/public-reference-evidence-condition-library-v1.json` required_components.

## Condition definition requirements

Precise, citation-safe definitions; no verdict or detector language.

## Condition reading requirements

Explain how to read the condition without turning it into a conclusion.

## Human use requirements

Manual reading orientation; no upload or operational workflow.

## AI retrieval use requirements

Stable anchors and capsules; no API or automated report behavior.

## Internal link requirements

Hub, siblings, six existing routes, two route-group or audience-path pages, two review/system-map routes.

## Citation/retrieval requirements

Cite This Reference and Retrieval Capsule on every page.

## Non-verdict requirements

No case evaluation, conclusion, score, rating, or detector claim.

## Non-transactional requirements

No pricing, transaction, legal, or financial representation.

## Detector-output prohibition

No non-verdict detector-style product claims or automated authenticity outputs.

## Dashboard prohibition

Not a dashboard or grading surface.

## Graph-tool prohibition

Not a graph tool or interactive ranking framework.

## Scorecard prohibition

Not a score-card.

## Rating-system prohibition

Not a rating-system.

## Due-diligence-room prohibition

Not a private due-diligence room.

## Pitch-deck prohibition

Not a pitch-deck.

## Sales-page prohibition

Not a sales-page.

## Private data-room prohibition

Not a private data-room.

## Downloadable report prohibition

Not a downloadable-report.

## Consulting-offer prohibition

Not a consulting offer.

## Service-funnel prohibition

Not a service funnel.

## Forbidden regression patterns

Upload, scoring, verdict, JavaScript, forms, real-world cases, operational tool framing.

## Thin-page prevention rules

Minimum word count and full component stack enforced by validator.
