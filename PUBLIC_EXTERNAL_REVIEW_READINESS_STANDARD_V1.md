# Public External Review Readiness Standard v1

## External Review Readiness Standard Statement

Public external-review readiness routes must help serious reviewers inspect Hoax.ai as a governed evidence-risk reference system without creating transaction behavior, detector claims, scoring, uploads, verdicts, APIs, automated reports, JavaScript, forms, chatbots, generators, or real-world case evaluation.

## Required routes

Five routes under `/external-review/` as defined in Sprint 101.

## Page structure requirements

Each route must include: Reference summary, Review purpose, Review path, What this page supports, What this page does not claim, Reference Answer, Source Confidence, Cite This Reference, Retrieval Capsule, page-end reference navigation, Boundary reminder, and Non-transactional review boundary.

## Depth requirements

Minimum 850 visible words per new route.

## Link requirements

- Homepage link to all five routes
- Each route links to `/external-review/`, at least 3 strategic routes, 5 reference/utility routes, and 2 sibling external-review routes

## Forbidden regression patterns

No pricing statements, transaction pages, representative mandates, legal or financial representations, interactive checklists, forms, or operational tool claims.

## Thin-page prevention

Routes below 850 visible words or missing required components fail validation.
