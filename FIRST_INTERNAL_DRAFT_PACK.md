# Hoax.ai First Internal Draft Pack

**Version:** v1.0.0  
**Status:** governed_internal_draft_pack  
**Maturity:** internal_drafts_only_no_routes_no_sitemap_no_publication  
**Decision:** DEC-039

## A. Purpose

This document introduces the first non-site-public internal draft files for Hoax.ai reference candidates.

## B. Non-Purpose

This internal draft pack does **not**:

- create public pages;
- create public routes;
- activate routes;
- expand sitemap.xml;
- create public metadata;
- create public navigation links;
- create SEO expansion;
- create structured data on public pages;
- create public classifier functionality;
- create a public tool;
- create upload functionality;
- create scoring;
- create forms;
- create analytics;
- deploy anything;
- enable DNS or Cloudflare;
- authorize publication.

## C. Governing Principle

**An internal draft may contain prose. It must not become a public surface.**

**The first drafts must prove reference discipline, not publication speed.**

## D. Internal Draft Definition

An internal draft is a repository-visible but non-site-public working reference artifact that contains draft prose under a blocked governance state. It is not a route, not a sitemap entry, not public navigation, not a public page, and not publication approval.
