# Public Reviewer Packet Standard v1

Sprint 102 — Public Reference Reviewer Packet v1  
**Decision:** DEC-120

## Reviewer Packet Standard Statement

Public reviewer-packet routes organize public inspection of Hoax.ai's reference surface. They must remain static, citation-safe, non-verdict, and non-transactional.

## Required page structure

1. Reference summary  
2. Packet purpose  
3. Review path (5–7 linked steps)  
4. What this page supports  
5. What this page does not claim  
6. Reference Answer  
7. Source Confidence  
8. Cite This Reference  
9. Retrieval Capsule  
10. Page-end reference navigation  
11. Boundary reminder  
12. Non-transactional review boundary  

## Required packet components

- Stable anchor IDs: reference-summary, packet-purpose, review-path, reference-answer, source-confidence, cite-this-reference, retrieval-capsule, boundary
- Canonical URL, title, meta description, Open Graph title and description
- At least 850 visible words of unique reviewer-packet content per route

## Review sequence requirements

- Seven-step static sequence as specified in Sprint 102
- Must state the sequence is not scoring and does not produce verdicts
- Links to entry points, narrative, utilities, pathways, readiness, and external review

## Public surface index requirements

- Eleven route groups aligned with current sitemap and route registry counts
- Representative links in every group
- No completeness claim beyond current public surface

## Citation/retrieval requirements

- Explain Cite This Reference, Retrieval Capsule, Reference Answer, and Source Confidence usage
- Prohibit citing Hoax.ai as detector, verdict source, pricing source, or transaction page

## Boundary summary requirements

- Reference-first posture, non-detector moat, inspectability as readiness
- External review as public inspection only

## Human readability requirements

- One H1 per page, clear section headings, linked review paths

## AI readability requirements

- Retrieval Capsules with canonical answers and boundary rules
- Stable anchors for agent retrieval

## Internal link requirements

- Link to homepage and /reviewer-packet/
- At least 3 strategic or external-review routes
- At least 5 reference, utility, or pathway routes
- At least 2 sibling reviewer-packet routes

## Non-verdict requirements

- No detector claims, verdict language as capability, scoring, or automated report promises

## Non-transactional requirements

- No pricing statements, transaction pages, acquisition term documents, representative mandates, legal or financial representation, private data-room access, or downloadable reports

## Forbidden regression patterns

Forbidden exact n-grams in public HTML include detector, upload, scoring, verdict, transaction, pricing, private data room, and downloadable report phrasing listed in Sprint 102 specification. Safe negative language is allowed.

## Thin-page prevention rules

- Minimum 850 visible words per new route
- All required sections and anchors must be present
