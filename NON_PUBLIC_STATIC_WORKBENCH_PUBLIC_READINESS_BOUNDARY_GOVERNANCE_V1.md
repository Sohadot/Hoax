# Non-Public Static Workbench Public-Readiness Boundary Governance v1

## Purpose

This document defines the boundary conditions for any future public-readiness discussion concerning the internal static Evidence Posture Workbench prototype.

## Non-Purpose

This sprint does not:

- make the prototype public-ready;
- create a public workbench;
- create a public route;
- create a sitemap entry;
- create public navigation;
- modify prototype files;
- create new prototype files;
- create an engine;
- create a classifier;
- create a detector;
- create upload functionality;
- create scoring;
- create fake/real outputs;
- create forms;
- create user inputs;
- create analytics;
- create an API;
- create monetization;
- modify DNS or Cloudflare;
- launch the Hoax.ai custom domain;
- authorize deployment.

## Governing Principle

Public-readiness boundary governance defines the conditions for future discussion. It does not authorize public release.

A public-ready boundary must protect users from mistaking a governed visual prototype for an operational detector.

## Public-Readiness Boundary Definition

Public-readiness boundary governance is a governance layer that defines the minimum prerequisites, risks, exclusions, route blockers, non-authorization rules, and future validation requirements that must exist before any future discussion of public exposure, public route creation, public navigation, sitemap eligibility, or public workbench status.

## Public-Readiness Is Not

Public-readiness is not:

- public launch;
- public route approval;
- sitemap approval;
- public navigation approval;
- engine approval;

- detector approval;

## Public Risk Boundary

Public risks that must be governed before any exposure:

- users may mistake the prototype for an operational detector;
- users may assume Not Assessable means fake;
- users may assume output envelope means conclusion;
- users may use the prototype to judge people, institutions, or events;
- users may expect upload/classification/scoring;
- public pages may imply product readiness;
- search snippets may misrepresent the workbench;
- public routing may imply launch;
- public navigation may imply tool availability;
- screenshots may detach prototype from non-operational context.

## Required Public Copy Boundary

Any future public-facing version must preserve:

- Framework, not verdict.
- Evidence posture, not truth verdict.
- Classify the evidence. Do not perform the verdict.
- Artifact risk does not transfer to subject guilt.
- Not Assessable is a protected governance state, not a conclusion.
- This is not an AI detector, fake/real classifier, upload tool, scoring tool, or verification service.

Do not add this copy to public pages in Sprint 42. Only define it as future required boundary.
