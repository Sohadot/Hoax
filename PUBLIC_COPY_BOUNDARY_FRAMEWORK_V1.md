# Public Copy Boundary Framework v1

## Public Copy Thesis

Hoax.ai must be useful without becoming a detector. Public copy defines evidence-risk utility language that helps people read evidence condition manually and conceptually — without upload workflows, verdict engines, fake/real classifiers, scoring systems, or automated report generators.

Public copy is not hidden governance. It is the governed language layer that makes Hoax.ai visible, legible, and practically useful while preserving non-verdict boundaries.

## Allowed Public Language

Public surfaces may describe Hoax.ai as:

- a public evidence-risk reference system
- a manual evidence posture guide
- a synthetic media risk literacy layer
- a non-verdict public utility
- a governed reference asset
- a manual evidence-risk utility
- guided evidence questions
- evidence posture reference
- non-verdict evidence map

Public surfaces may use utility framing:

- helps people read evidence risk
- helps people ask better evidence questions
- maps evidence risk manually and conceptually
- teaches when evidence becomes judgeable, or not
- does not judge artifacts for users

## Disallowed Public Language

Public surfaces must not claim or imply:

- detects fake media
- verifies truth
- scores authenticity
- proves manipulation
- identifies deception
- certifies evidence
- confirms real or fake
- automated investigation
- AI detector
- upload-and-scan workflow
- verdict engine
- truth machine as primary category headline

Boundary negations (e.g. “not a truth machine”) may appear in boundary sections but must not replace category-defining utility language in the hero.

## Detector-Misreading Risks

| Risk | Misread | Copy safeguard |
|------|---------|----------------|
| Detector framing | User expects fake/real output | Lead with evidence-risk utility, not detection |
| Upload framing | User expects file intake | No upload language; manual reference only |
| Score framing | User expects confidence percentage | No scoring language except explicit denial |
| Verdict framing | User expects true/false judgment | Non-verdict positioning in hero and sections |
| Tool hype | User expects live automated product | Manual, reference, and guided-question language |

## Public Utility Language

Approved utility verbs: guide, map, read, examine, reference, teach, ask, frame, structure.

Approved utility nouns: evidence risk, evidence posture, provenance, traceability, context, claim drift, boundary integrity, synthetic media literacy.

## Non-Verdict Positioning

Hoax.ai classifies evidence posture on artifacts — the condition of evidence before belief or action. Public copy must never collapse posture language into fake/real labels, guilt assignment, or truth certification.

## Homepage Copy Boundaries

The homepage hero uses evidence-first category language:

- **Headline:** Truth is no longer the first layer. Evidence is.
- **Subheadline:** Hoax.ai helps people read evidence risk in the synthetic media age — without issuing verdicts, scores, uploads, or fake/real labels.

The homepage must communicate usefulness, non-detector positioning, manual evidence mapping, and governed reference utility.

## Future Page Copy Boundaries

Future public utility pages (checklists, synthetic examples, posture maps, guides) must:

- remain manual and non-verdict
- avoid upload, API, scoring, and detector claims
- avoid real-world case evaluation of specific people, companies, political events, legal disputes, medical claims, or financial claims
- follow PUBLIC_UTILITY_LANGUAGE_STANDARD_V1.md and PUBLIC_DETECTOR_MISREADING_PREVENTION_V1.md
