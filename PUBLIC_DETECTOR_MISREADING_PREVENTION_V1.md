# Public Detector Misreading Prevention v1

## Prevention Statement

Hoax.ai public surfaces must prevent misreading as a detector, upload tool, truth machine, score engine, or verdict system while remaining useful as a manual evidence-risk utility.

## Detector Misreading Risks

- “analyze” without manual/reference context → live classifier
- “classify” without artifact-posture context → fake/real output
- “AI” adjacent to “media” → AI detector
- “check” without guided-question context → automated scan
- pillar language implying operational engine → product launch

**Safeguard:** Lead with manual evidence-risk utility; pair “classify evidence posture” with artifact-only and non-verdict qualifiers.

## Upload-Tool Misreading Risks

- “submit”, “upload”, “drop your file”, form elements
- implied file intake or URL paste workflow

**Safeguard:** No forms, no upload copy, no file-input language on public surfaces.

## Truth-Machine Misreading Risks

- hero negation-only positioning
- “verify truth”, “confirm real or fake”, “certifies evidence”
- truth score or authenticity percentage language

**Safeguard:** Evidence-first hero; explicit non-verdict subheadline; boundary section for truth-machine denial.

## Score-Engine Misreading Risks

- confidence percentage, authenticity score, risk score as product output
- ranking or grading language

**Safeguard:** No scoring language except explicit denial; posture states instead of numeric scores.

## Verdict-Engine Misreading Risks

- fake/real labels, guilty/innocent framing, manipulation proof claims
- subject accusation or event determination

**Safeguard:** Artifact–subject separation; posture language; guided questions not verdicts.

## Public Copy Safeguards

- Utility-first hero and subheadline
- “What Hoax.ai is not” boundary section
- Manual and reference qualifiers on utility claims
- Forbidden language list in PUBLIC_UTILITY_LANGUAGE_STANDARD_V1.md
- Validator enforcement on homepage copy

## Interface Safeguards

- No JavaScript-driven interactive classifier UI
- No forms, uploads, or API endpoints on public pages
- No result cards, score displays, or verdict badges
- HTML/CSS reference and map previews only

## Future Public Utility Safeguards

Future authorized utilities (manual checklists, synthetic examples, posture maps, guides) must:

- remain non-upload and non-automated
- use guided questions not verdict output
- avoid real-world evaluation of specific people, companies, political events, legal disputes, medical claims, or financial claims
- pass copy boundary validator before publication
