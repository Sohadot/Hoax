# Public Audience Path Standard v1

Sprint 113 — Public Reference Audience Path Expansion v1  
**Decision:** DEC-131

## Audience Path Standard Statement

Audience-path pages explain safe public reading and retrieval paths for major reader types without creating personas, marketing funnels, consulting offers, service funnels, dashboards, graph tools, scorecards, rating systems, due-diligence rooms, private data-room access, downloadable reports, pitch decks, sales pages, upload behavior, scoring, verdicts, detector behavior, or automated reports.

## Required page structure

Reference summary, conceptual thesis, contextual frame, audience path purpose, audience route map, human use, AI retrieval use, confusion this path prevents, what this path supports, what this path does not claim, reference answer, source confidence, cite this reference, retrieval capsule, strategic reference value, page-end reference navigation, boundary reminder, non-transactional review boundary.

## Minimum depth requirement

At least 1,400 visible words per audience-path page.

## Required reference components

reference_summary, conceptual_thesis, contextual_frame, audience_path_purpose, audience_route_map, human_use, ai_retrieval_use, confusion_this_path_prevents, reference_answer, source_confidence, cite_this_reference, retrieval_capsule, strategic_reference_value, page_end_reference_navigation, boundary_reminder, non_transactional_review_boundary.

## Audience route-map requirements

Each page must link representative routes in a ordered reading path for its audience.

## Human use requirements

Explain ordered reading discipline and what the path must not be used to infer.

## AI retrieval use requirements

Explain safe citation and boundary preservation — not operational authority or API behavior.

## Internal link requirements

Each page links to homepage, `/system-map/`, `/audience-paths/`, at least two sibling audience-path pages, at least six existing reference routes, at least two route-group deepening pages, and at least two review/overview/system-map routes.

## Citation/retrieval requirements

Stable anchor IDs, Reference Answer blocks, Source Confidence blocks, Cite This Reference blocks, and Retrieval Capsules on every page.

## Non-verdict requirements

No case evaluation, conclusion, score, rating, detector output, or automated report promise.

## Non-transactional requirements

No pricing, transaction, acquisition term, representative mandate, legal representation, or financial representation.

## Consulting-offer prohibition

No consulting offer or professional services funnel language.

## Service-funnel prohibition

No lead-generation, signup funnel, or service funnel behavior.

## Dashboard prohibition

No interactive dashboards, analytics panels, or rating widgets.

## Graph-tool prohibition

No graph visualization tools or interactive network maps.

## Scorecard prohibition

No scorecards or graded surface summaries.

## Rating-system prohibition

No rating systems or numeric certainty outputs.

## Due-diligence-room prohibition

No private due-diligence room framing.

## Pitch-deck prohibition

No pitch-deck behavior or downloadable deck claims.

## Sales-page prohibition

No sales-page behavior or purchase framing.

## Private data-room prohibition

No private data-room access claims.

## Downloadable report prohibition

No downloadable report behavior.

## Forbidden regression patterns

Upload fields, forms, JavaScript interaction, public API behavior, chatbot/generator behavior, real-world case evaluation, forbidden positive promotional phrases in public HTML.

## Thin-page prevention rules

No generic persona pages, no marketing copy, no filler; each page must have a unique conceptual thesis native to Hoax.ai.
