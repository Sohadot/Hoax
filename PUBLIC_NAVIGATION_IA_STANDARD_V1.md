# Public Navigation IA Standard v1

## Navigation IA Standard Statement

Public navigation must reveal Hoax.ai’s evidence-risk system structure through route groups, page roles, system navigation blocks, and IA Capsules — without implying automated assessment.

## Required Homepage Components

- Navigate Hoax.ai by Evidence-Risk Layer section
- Five route groups with role explanations and Start here if lines
- Hoax.ai System Navigation block

## Required Page Components (22 utility/reference/pathway pages)

- Visible page role label near H1
- Hoax.ai System Navigation block
- IA Capsule with page role, route group, primary concept, upstream, downstream, related pathway, boundary rule
- Page-specific Where to go next section

## Page Role Labels

Allowed: Public Utility, Core Reference Concept, Deep Reference Concept, Evidence-Risk Pathway, Boundary Reference, Standard Reference, Governance Reference.

## System Navigation Requirements

Five IA groups with concise selected links — not unstructured 41-link dumps.

## IA Capsule Requirements

Static HTML only. No JSON-LD. No JavaScript.

## Pathway Entry Points

Key reference pages must link to matching pathway routes.

## Boundary Requirements

Navigation must not imply upload, scoring, verdict, detector, API, or report behavior.

## Forbidden Patterns

Forbidden exact n-grams per Sprint 94 governance.

## Thin-Navigation Prevention

Each Where to go next section must be page-specific, not identical boilerplate.
